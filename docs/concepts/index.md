# Concepts

The Concepts section defines the foundational vocabulary and design principles of agentic-lab.
Every term introduced elsewhere in the documentation links back to a definition here.

---

## In this section

| Page | Description |
|------|-------------|
| [ASL](asl.md) | The Agentic Specification Language — the YAML-based format that encodes your entire multi-agent system |
| [MABaC](mabac.md) | Multi-Agent Behavior as Code — behavioral metadata that travels with agent code |
| [Governance Contract](governance-contract.md) | The version-controlled artifact that replaces opaque prompt-based safety with auditable policy |
| [Agent Card](agent-card.md) | Per-agent identity specification analogous to a digital certificate |
| [Behavioral Envelope](behavioral-envelope.md) | The deterministic enforcement boundary around a stochastic agent |
| [Authorization](authorization.md) | ASL's tool-binding authorization model and its relationship to IAM/RBAC |

---

!!! tip "Maturity legend"
    Throughout the Concepts section, pages are labelled with one of four maturity levels:

    - **`[IMPLEMENTED]`** — shipped in the current release
    - **`[PILOT-VALIDATED]`** — tested in a real deployment, not yet stabilised
    - **`[IN DEVELOPMENT]`** — actively being built
    - **`[RESEARCH]`** — working draft for community feedback
